dockerfile的基本结构

Dockerfile 一般分为四部分：

  1. 基础镜像信息

  2. 维护者信息

  3. 镜像操作指令

  4. 容器启动时执行指令

’#’ 为 Dockerfile 中的注释

FROM

指定基于哪个基础镜像,必须为第一个命令

MAINTAINER

维护者信息

RUN

构建镜像时执行的命令

CMD

构建容器后调用，也就是在容器启动时才进行调用

COPY

复制文件到镜像中

使用copy命令时,必须将需要copy的文件放在和dockerfile同一目录下

ENTRYPOINT

配置容器，容器启动时要执行的命令，它和CMD很像，也是只有一条生效

和CMD不同是：CMD 是可以被 docker run 指令覆盖的，而ENTRYPOINT不能覆盖

LABEL

用于为镜像添加元数据

EVN

设置环境变量,它主要是为后续的RUN指令提供一个环境变量

EXPOSE

VOLUME

用于指定持久化目录

WORKDIR

工作目录,类似于cd命令
